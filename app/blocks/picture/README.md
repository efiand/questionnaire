# Шаблон адаптивного изображения (макрос)

Создаёт разметку блока `<picture>` с кадрированием, ретинизацией и поддержкой webp (предполагается, что данный таск используется в сборке).

## Объявление в основном шаблоне

```twig
{% from "source/blocks/_picture/picture.njk" import picture with context %}
```

## Вызов

```twig
{{ picture(params) }}
```

## Структура входного объекта

```json
{
  "src": "img/logo.png",
  "mq": [
    "desktop",
    "tablet"
  ],
  "alt": "Логотип",
  "classes": "header__logo",
  "imgClasses": "header__logo-img",
  "nowebp": true,
  "retina": [2, 4]
}
```

Все ключи, кроме, `img`, носят опциональный характер.

* `mq` – перечень алисов медиазапросов для кадрирования (алиасы задаются глобально через словарь `mq` в настройках шаблона). Для корректной работы у соответствующих изображений перед расширениями должны быть суффиксы вида `-<alias>`.
* `alt` – значение атрибута `alt` тега `<img>`.
* `classes` – строка с БЭМ-миксами всего контейнера (в зависимости от того, куда встраивается блок `.picture`).
* `imgClasses` – строка с БЭМ-миксами непосредственно `<img>` (в зависимости от того, куда встраивается блок `.picture`).
* `nowebp` – при истинном значении не создаются `<srcset>` для webp-версий. Уместно для svg-изображений.
* `retina` – перечень значений плотностей пикселей.
